## 🔧 Technical Details

### Why Version Mismatch Causes "Not Installed" Error

n8n's node loading system:
1. Workflow requests: `n8n-nodes-base.googleCalendar` with `typeVersion: 2`
2. n8n searches registry for: `googleCalendar` version 2
3. Only finds: `googleCalendar` version 1.0
4. Conclusion: "This node is not currently installed"
5. Result: Execution stops, workflow fails

### Downstream Impact

**Nodes That Never Execute**:
1. ❌ Create Google Calendar Event (stops here)
2. ❌ Update Appointment (never reached)
3. ❌ Create Appointment Reminders (never reached)
4. ❌ Send Confirmation Email (never reached)

**User Impact**:
- No Google Calendar events created
- Appointment status remains `pendente` (not updated to `confirmado`)
- No reminder tasks created
- No confirmation emails sent
- Complete workflow failure from user perspective

## 🎓 Lessons Learned

### Node Version Management
- **Always verify installed node versions** before creating workflows
- **Check n8n node registry** (`*.node.json` files) for `nodeVersion`
- **Match workflow typeVersion** to installed node capabilities
- **Test workflows after n8n upgrades** (node versions may change)

### Error Message Interpretation
- **"Install this node"** doesn't always mean node is missing
- **Could indicate version mismatch** between workflow and installation
- **Verify node exists in filesystem** before assuming installation issue
- **Check node registration files** for accurate version information

### Workflow Validation
- **Test complete end-to-end flow** before marking production-ready
- **Verify ALL downstream nodes execute** (not just primary nodes)
- **Check execution logs** for subtle failures in later steps
- **User reports are critical** - they expose real-world issues
