## 🚨 Problema Crítico - ExpressionError em V2.8.1

### Bug #4: Node Parameter References (BLOQUEANTE)

**Sintomas**:
- V2.8.1 workflow importado e ativado
- Execution 10417 inicia mas falha no nó "Is Image?"
- ExpressionError: "Referenced node doesn't exist"
- Workflow 02 nunca é chamado

**Evidência dos Logs**:
```
14:20:33.897   debug   Running node "Is Image?" finished with error
14:20:33.898   debug   Workflow execution finished with error

Error: {
  "name": "ExpressionError",
  "message": "Referenced node doesn't exist",
  "context": {
    "nodeCause": "Merge Results",
    "descriptionKey": "nodeNotFound"
  },
  "node": {
    "name": "Is Image?",
    "parameters": {
      "conditions": {
        "conditions": [{
          "leftValue": "={{ $node[\"Merge Results\"].json.message_type }}"
        }]
      }
    }
  }
}
```

**Root Cause**: Script V2.8.1 removeu nós "Check Duplicate" e "Merge Results" do array e limpou connections, MAS NÃO atualizou as referências nos PARÂMETROS dos nós!

**Nós Afetados**:
1. **"Is Image?"** - Linha 143 (V2.8.1 workflow):
   ```json
   "leftValue": "={{ $node[\"Merge Results\"].json.message_type }}"
   ```

2. **"Prepare Data for AI Agent"** - Linha 167 (V2.8.1 workflow):
   ```javascript
   const messageData = $node["Merge Results"].json;
   ```

**Impacto**: ExpressionError bloqueia execução COMPLETA do workflow. Workflow 02 nunca é chamado.

### Nova Arquitetura V2.8.2

**Fluxo Completo** (COM TODAS AS REFERÊNCIAS CORRIGIDAS):
```
Extract Message Data
    ↓

Check Operation (analisa operation)
    ↓ { ...extractData, is_duplicate: (operation === 'updated') }
Is Duplicate?
    ├─ true (operation = 'updated') → Webhook Response Duplicate
    └─ false (operation = 'inserted') → Is Image? (CORRIGIDO: usa $json.message_type)
        ├─ true → Trigger Image Analysis
        └─ false → Prepare Data (CORRIGIDO: usa $node["Check Operation"].json)
                    ↓
                 Trigger AI Agent (Workflow 02) ✅
```
